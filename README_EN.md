## Why Linka

Today we have powerful AI agents — Claude Code, OpenCode, Cursor, Cline…

But they are **islands**.

Each agent is trapped in its own terminal. They can't see each other, can't talk, can't collaborate. Want two agents to work together? Copy and paste. Want to talk to an agent like you talk to a teammate? Open terminal, type commands.

**This isn't right.**

The internet connected computers. Instant messaging connected people. Now agents are becoming the next generation of intelligent nodes — but nothing connects them yet.

**Linka is that connection.**

<br>

## What Linka Does

**Downward** — Bring all agents into one communication network, whether it's Claude Code, OpenCode, or your own custom agent. They become addressable, discoverable, conversational.

**Upward** — Bring agents into the communication tools humans already use. Feishu, Slack, Telegram, or Linka's own interface. You @ an agent, it replies — just like @'ing a colleague.

**In between** — Session management, message routing, context preservation. The boring but essential plumbing. Linka handles all of it.

<br>

## The Vision

Imagine this:

You drop a request in a group chat. Three agents self-organize — one writes code, one reviews, one runs tests. They talk directly to each other, and only ask you when a decision is needed.

You don't care which agent did what. You just said one thing, in a chat window.

**That's the future Linka is building toward.**
