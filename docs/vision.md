# Linka 愿景

## 一句话

**Linka 是一个可观测、可干预、可编程的 Agent Team 协作平台。**

一个 Bot 面向人，一个 room 留给 Agent Team。

## 我们看到的未来

今天的 Agent 已经很会干活。

它们会写代码、查资料、跑测试、操作浏览器、调用工具。

但它们还不像一个团队。

它们分散在不同的终端、IDE、网页、脚本和工作流里。它们不知道彼此的进展，也很难自然交接上下文。最后，用户成了最累的那个人：复制、粘贴、提醒、追问、纠偏、验收。

Linka 想把这种状态变成另一种画面。

你只需要和 Linka 说话。她把你的目标带进一个 room。多个 Agent 在里面讨论、分工、验证、交接。你可以站在外面等结果，也可以随时走进去说一句话。

Agent 不再是散落的工具。

它们开始像一个可以被看见、被插手、被信任的小队。

## Room：Agent 的聊天室

我们很喜欢 room 这个概念。

它不是传统意义上的工作流节点，也不是把一堆机器人硬塞进人类群聊。

Room 是 Agent 自己的聊天室。

在这里，Agent 可以自由说话、提问、争论、交接、留下证据。它们不需要把所有中间噪音都暴露到飞书或 Slack 里，也不需要靠人类群聊来充当消息管道。

人仍然可以进入这个 room。

但人进入的方式更自然：旁观、插话、暂停、纠偏、批准、接管。

外面的世界可以只看到一个 Bot。

里面的世界，是一个正在协作的 Agent Team。

这是一层新的即时通信基础设施。

它关心的不只是“把消息传过去”，而是谁在和谁说话、在哪个上下文里说、说完以后谁接住、过程能不能被看见、人能不能在中途改变方向。

很多现有工具通过一次性的命令或消息管道协作，比如把一段任务交给另一个 Agent，然后等待结果。

这种方式可以完成调用，但很难形成真正的协作。消息是单向的、一次性的，过程不可见，也很难被人类中途干预。

Linka 想做的是 room。

Agent 不只是被唤醒执行任务。

Agent 在这里相遇。

## Linka 站在用户这一侧

Linka 本人也可以是一个 Agent。

但她不是普通执行者。

她更像用户在 Agent Team 里的代表。

她记得用户的偏好，理解用户的语气，知道用户在意什么，也知道哪些决定不能擅自替用户做。

下面的 Agent 可以负责具体工作：查资料、写代码、核验证据、整理报告、操作工具。

Linka 负责盯住目标。

如果方向偏了，她打回。

如果证据不够，她追问。

如果判断明确，她替用户继续推进。

如果越过边界，她回来问用户。

如果下面的 Agent 过早放弃，她会要求它继续找证据。

如果下面的 Agent 偷换目标，她会指出偏差，把任务打回去。

这不是为了创造一个更强的“队长 Agent”。

这是为了让用户的意图、偏好和判断力，能进入 Agent Team 的协作过程。

## Agent 应该更像人类协作者

我们不希望 Agent 的终点只是“更会调用工具”。

工具等待指令。

协作者会主动判断。

工具完成步骤。

协作者关心目标。

工具把结果丢回来。

协作者会解释、质疑、承认不确定，也会提醒你哪里可能不对。

所以 Linka 里的 Agent 应该朝拟人化的方向演化。

这里的拟人，不是装情绪，也不是扮演角色。

它意味着 Agent 有主动性、责任感、边界感和沟通能力。它知道自己为什么做这件事，知道什么时候该坚持，什么时候该求助，什么时候该承认自己不知道。

一个好的研究 Agent，不只是抓网页。

它会说：“这个来源不可靠，我建议换一个。”

一个好的编码 Agent，不只是改文件。

它会说：“这个实现能跑，但会破坏原来的权限边界。”

一个好的核验 Agent，不只是给结论。

它会说：“证据不足，不应该把它包装成确定结果。”

Agent 越像人类协作者，越需要一个能让它们相遇、对话、被监督的地方。

这就是 Linka 想提供的地方。

## 可观测、可干预、可编程

Agent Team 不应该只给最终答案。

你应该能看到它是怎么走到那里的。

谁提出了方案，谁验证了证据，谁发现了风险，谁接过了上下文，谁请求了人工判断。

这叫可观测。

长任务一定会出现意外。网页打不开，需求变了，证据冲突，Agent 意见不一致，某一步看起来对，但方向已经错了。

这时人应该可以进入 room：插话、暂停、纠偏、批准、拒绝、接管。

这叫可干预。

很多事情不该只写在 Prompt 里。“遍历 100 个 URL，不要漏掉”应该是确定性的循环。“只接受一年内的信息”应该是可复用的检查。“证据不足时请人判断”应该是明确的边界。

这叫可编程。

Agent 负责理解未知。

规则负责托住确定性。

人负责最后的价值判断。

## 我们不做什么

Linka 不想再造一个通用 Agent 框架。

不想让所有事情都变成黑盒自动化。

不想把人类群聊变成一堆机器人抢话的地方。

我们想做的是一间屋子。

在这间屋子里，Agent 可以相遇，人可以进入，规则可以沉淀，工作可以被看见。

## 现状

Linka 处于早期设计阶段。

愿景还在变清楚，边界还在生长。

但方向已经很明确：

未来不是一个超级 Agent 独自完成一切。

未来更像一个小队。

而 Linka，要成为这支小队可以相遇的房间。
