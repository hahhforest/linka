# Linka 愿景

## 一句话

**Linka 是 Agent 时代的通信基础设施。**

互联网连接了计算机。即时通讯连接了人。Linka 连接 Agent。

## 我们看到的问题

今天的 AI Agent 已经很强大了。但它们是孤岛。

每个 Agent 独立运行在自己的终端或 IDE 里，互相看不见，无法对话，无法协作。你想让两个 Agent 配合完成一件事？手动复制粘贴上下文。你想像跟同事聊天一样跟 Agent 说话？打开终端，敲命令。

Agent 在能力上已经是"同事"了，但在沟通方式上还停留在"命令行工具"。

这不对。

## 核心信念

**Agent 是通信实体，不是工具。**

你不"调用"同事。你和同事对话。Agent 应该有身份、有地址、能被发现、能主动发起对话。它们应该是通信网络中的一等公民。

**人与 Agent 共享同一个通信平面。**

人类已经生活在消息流里——飞书、Slack、Telegram。Agent 不需要发明新的交互方式，它们应该加入人类已有的对话空间。你 @ 一个 Agent，它回复你，就像 @ 同事一样自然。

**连接产生涌现。**

单个 Agent 的能力是有限的。但当 Agent 能自由地发现彼此、委托任务、交换上下文时，系统的能力将超越所有个体之和。这不是编排——编排需要一个全知的指挥者。这是通信——去中心化的、自组织的、涌现的。

## Linka 做什么

Linka 是一个协议级的通信层，坐在 Agent 和人类之间。

**向下**，它为各种 Agent 提供统一的通信接口。无论是什么框架、什么模型，接入 Linka 后就可以被寻址、被发现、与其他 Agent 对话和委托。

**向上**，它把 Agent 带进人类已有的沟通方式里。通过 IM 桥接或 Linka 自己的界面，用户用最自然的方式——聊天——与 Agent 交互。

**中间**，它处理所有脏活累活：会话生命周期、消息路由、上下文接力、多端同步。

## 终极图景

想象这样的世界：

你在群聊里说了一句需求。三个 Agent 自动分工——一个写代码、一个审查、一个跑测试。它们之间直接沟通，需要你决策时才来问你。

一个新的 Agent 上线后，它向网络注册自己的能力，其他 Agent 可以发现并与它协作——无需任何人手动配置。

你不关心是哪个 Agent 完成的。你只是在聊天窗口里，说了一句话。

## 我们不做什么

Linka 不做 Agent 本身。不做模型推理，不做代码生成，不做任何"智能"。

Linka 做的是 Agent 之间的空气——让信号能够传递的介质，让对话能够发生的空间。

## 多端策略

Linka 的目标是人人可用：

- **桌面应用**（macOS / Windows）：一键安装，双击启动。普通用户不需要懂终端。
- **Web 界面**：浏览器访问，管理和交互。
- **命令行**：`linka start`。专业用户和服务器部署的入口。
- **IM 桥接**：飞书、Slack、Telegram——在你已有的聊天工具里直接使用。

所有端共享同一个内核。桌面 App、Web UI、CLI 只是同一个通信枢纽的不同皮肤。

## 宏观架构

```
用户层：  Desktop App  |  Web UI  |  CLI  |  IM (飞书/Slack/...)
              │             │         │           │
              └─────────────┴────┬────┴───────────┘
                                 │
                          Linka Protocol
                                 │
                          ┌──────┴──────┐
                          │ Linka Daemon │
                          └──────┬──────┘
                                 │
                          Adapter Layer
                                 │
                    ┌────────┬───┴───┬────────┐
                    │        │       │        │
                 Agent A  Agent B  Agent C   ...
```

核心是 **Daemon**——一个本地运行的通信枢纽。所有端（桌面、Web、CLI、IM）都是它的客户端。所有 Agent 通过适配器接入它。

技术选型方向：
- **Monorepo**（pnpm + Turborepo），所有包在一个仓库
- **桌面端 Tauri v2**，包体小、原生性能，daemon 作为 sidecar 进程
- **Web UI 复用**：桌面 App 和浏览器加载同一套 React SPA

## 现状

Linka 处于早期设计阶段。愿景已定，架构方向已对齐，正在从协议层开始逐步构建。
